## Slide 2: Project Objectives
- Build one platform for food, travel, and expense recommendations.
- Personalize results using cuisine, dietary needs, budget, and travel interests.
- Track expenses and guide users toward budget-friendly trip decisions.
- Provide explainable recommendations instead of black-box suggestions.
- Deliver the solution through a Python engine, Flask API, and HTML frontend.

## Slide 3: Problem Statement
- Trip planning is fragmented across separate food, travel, and budgeting tools.
- Generic suggestions often ignore user preferences, dietary restrictions, and group size.
- Manual planning increases time, confusion, and the risk of overspending.
- Users need a single smart system that recommends options and monitors expenses together.

## Slide 4: Project Overview - Introduction
- ExpenseMate AI is a Python-based recommendation platform for restaurants, destinations, and budgets.
- It loads structured CSV datasets and converts user preferences into ranked recommendations.
- The system combines food discovery, travel planning, and expense analysis in one workflow.
- A demo app, Flask backend, and browser frontend make the project easy to present and extend.

## Slide 5: End User
- Students and young travelers planning affordable trips.
- Families, couples, and groups comparing destination and dining options.
- Budget-conscious users who want spending alerts and category-wise tracking.
- Users with cuisine preferences or dietary restrictions seeking better matches.
- Developers and institutions using the project as a capstone or recommendation-system demo.

## Slide 6: Wow Factor
- One project solves three real problems: food choice, destination choice, and budget control.
- Recommendations are explainable, so users can see why each option was selected.
- Combo trip planning merges restaurant and destination suggestions into one package.
- Budget logic adds spending summaries, percentage usage, and practical advice.
- The project already includes a working API and interactive frontend for live demonstration.

## Slide 7: Modelling
1. Input: user profile captures cuisine, dietary, budget, travel, and group preferences.
2. Data layer loads 10,000 restaurants, 138 destinations, and 2,000 expense records.
3. Food and travel recommenders score each item using similarity rules and ratings.
4. Expense recommender computes spending summaries and budget recommendations.
5. Combo recommender merges outputs into a complete trip package with cost estimation.
6. Flask API sends results to the HTML frontend for user interaction.

## Slide 8: Result / Outcomes
- The system successfully loads a large local dataset and produces personalized recommendations.
- Current support includes 20 cuisines, 10 travel categories, and 8 expense categories.
- Users can explore food options, travel options, and budget allocations from one interface.
- Budget tracking reports spent amount, remaining balance, daily average, and category breakdown.
- The project is ready for demo, further research, and future real-world integration.

## Slide 9: Conclusion
- ExpenseMate AI demonstrates how recommendation logic can improve planning decisions.
- The project is modular, explainable, and practical for real travel and dining use cases.
- By combining recommendations with expense monitoring, it delivers more complete user support.
- This capstone provides a strong base for smarter, data-driven personal assistant systems.
